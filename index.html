CI/CD Pipeline Project using Jenkins and AWS EC2

Project Overview

This project demonstrates a fully automated CI/CD pipeline built using Jenkins, GitHub, and AWS EC2. Each time code is pushed to the main branch, Jenkins automatically builds, tests, and deploys the latest version of the application to an EC2 instance.

Pipeline Workflow

  1. GitHub Commit: Changes are committed and pushed to GitHub, triggering a Jenkins build automatically via a webhook.
  2. Jenkins Build: Jenkins pulls the latest code, runs automated steps defined in the Jenkinsfile, and prepares the deployment package.
  3. Deployment to EC2: The latest build is deployed on the EC2 server, replacing the previous version and keeping the application up to date.

Technologies Used

  • Jenkins for automation
  • GitHub for version control
  • AWS EC2 for hosting
  • Linux (Ubuntu) for server management
